# Claude Task Template

Use this structure for every Claude Code task.

## Objective
What this task builds.

## Why this matters
Why the task exists and what it unlocks next.

## Scope
Exactly what is included.

## Out of scope
What must not be changed or invented here.

## Business rules to respect
Concrete business rules from the source docs.

## Files expected to change
Expected areas of the repo.

## Acceptance criteria
Clear observable outcomes.

## Required automated checks
- pnpm lint
- pnpm typecheck
- pnpm test
- pnpm build
- pnpm test:e2e when relevant

## Manual QA checklist
Simple user-facing checks.

## Stop and ask if
Ambiguities or business-rule conflicts.
