# Task 05.1 — Billing concept schema

## Objective
Create the billing concept data model.

## Scope
- BillingConcept model
- source types
- statuses
- recurrence support for manual recurring billables
- relation to contract line where applicable
- no invoice model

## Out of scope
- generation engine
- UI

## Acceptance criteria
- Schema supports automatic and manual billing concepts
- No invoice system introduced

## Stop and ask if
- recurrence structure becomes too generic for phase 1
