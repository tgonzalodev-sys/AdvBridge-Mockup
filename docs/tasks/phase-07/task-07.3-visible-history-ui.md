# Task 07.3 — Visible history UI refinement

## Objective
Make the visible history in Madre detail useful enough for phase 1.

## Scope
- present old/new value changes clearly
- emphasize commercial evolution:
  - prices
  - licenses
  - similar meaningful changes
- keep other history stored but not prominently exposed

## Out of scope
- global history screen
- contract-specific history UI

## Acceptance criteria
- Madre detail history is understandable and useful
- no dashboard/reporting scope creep

## Stop and ask if
- the stored history format makes UI interpretation too noisy
