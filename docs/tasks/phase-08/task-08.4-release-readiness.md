# Task 08.4 — Release readiness

## Objective
Leave the app in a stable, reviewable state.

## Scope
- final lint/typecheck/test/build pass
- doc sync check
- remove dead code
- sanity review against acceptance criteria
- summarize known limitations

## Out of scope
- new features
- future modules

## Acceptance criteria
- Repo passes checks cleanly
- Docs and implementation still match
- Known limitations are documented

## Stop and ask if
- uncovered business ambiguity remains
