# Task 06.1 — Billing matrix UI

## Objective
Build the main yearly billing matrix screen.

## Scope
- subsidiary selector
- year selector
- filters
- Madre rows
- month columns
- total + warning indicator
- mini totals by currency
- open monthly detail drawer

## Out of scope
- invoiced marking logic
- drawer internals beyond navigation shell

## Acceptance criteria
- Matrix is usable and readable
- subsidiary context is respected
- mixed currencies are not visually combined

## Stop and ask if
- performance requires pagination or virtualization decisions
