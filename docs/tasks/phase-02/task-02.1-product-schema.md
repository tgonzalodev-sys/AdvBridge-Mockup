# Task 02.1 — Product schema

## Objective
Create the product and version data model.

## Scope
- Product model
- ProductVersion model
- category enum/reference
- fee component flags
- usesLicenses
- availableForNewContracts
- uniqueness on product name

## Out of scope
- UI
- contract usage rules

## Acceptance criteria
- Product and versions are modeled cleanly
- Products can have zero or many versions

## Stop and ask if
- Version storage design becomes overcomplicated
